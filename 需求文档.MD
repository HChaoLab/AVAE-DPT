

# 需求文档：基于对抗变分自编码器的原发-转移拟时序推断模型 (AVAE-DPT)

## 1. 项目背景与目标

本项目旨在分析多个病人的单细胞转录组数据（scRNA-seq），构建从“原发部位”到“转移部位”的拟时序（Pseudotime）轨迹。

**核心挑战**：直接降维或批次校正会导致潜变量空间被病人的个体差异（批次效应）主导，而非生物学演化过程。

**解决方案**：构建一个带有对抗判别器和条件约束的变分自编码器（Adversarial CVAE / DANN-VAE）。在潜变量空间 $z$ 中，通过“对抗学习”剔除病人特异性信号，同时通过“监督学习”强化原发到转移的演化特征。

## 2. 数据输入与结构

- **表达量矩阵** $X$：经过预处理的高变基因（HVG）表达矩阵（建议支持原始 count 并用 NB loss，或对数标准化后的数据用 MSE loss）。
- **批次标签** $Y_{batch}$：病人的 ID（离散型，建议转为 One-hot 编码）。
- **阶段标签** $Y_{stage}$：原发（Primary, 标为0）与转移（Metastasis, 标为1）（二分类标签）。

## 3. 模型架构设计 (四大模块)

请使用 PyTorch 实现以下四个主要网络模块（基于 MLP）：

1. **编码器 (Encoder)**
   - **输入**：表达矩阵 $X$。
   - **输出**：潜变量分布的均值 $\mu$ 和对数方差 $\log\sigma^2$。通过重参数化技巧（Reparameterization trick）采样得到潜变量 $z$。
2. **解码器 (Decoder)**
   - **输入**：潜变量 $z$ 与批次标签 $Y_{batch}$ 的拼接（Concatenation）。
   - **输出**：重构的表达矩阵 $\hat{X}$。
   - *设计意图*：将批次信息作为条件（Condition）输入给解码器，使编码器无需在 $z$ 中保留批次信息即可完成重构。
3. **阶段预测器 (Stage Predictor)**
   - **输入**：潜变量 $z$。
   - **输出**：预测细胞阶段的概率（Sigmoid 激活，输出 $\hat{Y}_{stage}$）。
4. **病人辨别器 (Patient Discriminator) + GRL**
   - **输入**：潜变量 $z$。在输入该网络前，**必须经过一个梯度反转层 (Gradient Reversal Layer, GRL)**。
   - **输出**：预测病人归属的概率分布（Softmax 激活，输出 $\hat{Y}_{batch}$）。

## 4. 损失函数定义

模型在反向传播时，需要联合优化以下四个损失组件：

$$\mathcal{L}_{Total} = \mathcal{L}_{Recon} + \beta \mathcal{L}_{KL} + \gamma \mathcal{L}_{Stage} + \lambda \mathcal{L}_{Patient}$$

*注意：由于病人辨别器前端加了 GRL，在代码实现时 $\mathcal{L}_{Patient}$ 直接相加即可，GRL 会自动在反向传播时将该部分的梯度反转（乘以 -1），从而实现极大极小博弈（Minimax game）。*

- **$\mathcal{L}_{Recon}$ (Reconstruction Loss)**：$X$ 与 $\hat{X}$ 之间的重建损失（NB ）。
- **$\mathcal{L}_{KL}$ (KL Divergence)**：潜变量后验分布与标准高斯先验 $\mathcal{N}(0, I)$ 之间的 KL 散度。
- **$\mathcal{L}_{Stage}$ (Stage Classification Loss)**：$Y_{stage}$ 与 $\hat{Y}_{stage}$ 之间的二元交叉熵损失（BCE Loss）。
- **$\mathcal{L}_{Patient}$ (Patient Adversarial Loss)**：$Y_{batch}$ 与 $\hat{Y}_{batch}$ 之间的多类别交叉熵损失（CrossEntropy Loss）。

## 5. 推理与拟时序提取 (Inference & Pseudotime)

请在代码中实现一个单独的推断函数（例如 `get_pseudotime(X)`），执行以下逻辑：

1. 将细胞表达数据 $X$ 输入 **Encoder**，获取均值 $\mu$ 作为细胞在潜空间的确定性表征。
2. 将 $\mu$ 输入 **Stage Predictor**，获取阶段预测的概率值（Logit 或 0~1 的 Sigmoid 输出）。
3. 直接将该输出值作为**转移拟时序分数 (Metastasis Pseudotime Score)** 返回。

## 6. 给代码助手的实现要求 (Claude Code Directives)

1. **GRL 实现**：请使用 `torch.autograd.Function` 自定义实现 Gradient Reversal Layer。要求支持动态调整反转系数 $\lambda$（在训练初期 $\lambda$ 较小，随着 Epoch 增加逐渐升至 1，以保证训练初期的稳定性）。
2. **超参数配置**：请将隐层维度、潜空间维度（如 `z_dim=20`）、各 Loss 的权重因子（$\beta, \gamma, \lambda$）解耦并放在独立的 config 字典或类中，方便后续调参。
3. **数据接口**：请提供一个基于 `anndata.AnnData` 的 Dataset/DataLoader 封装示例，演示如何将 AnnData 中的 `X`, `obs['patient_id']`, `obs['stage']` 喂给模型。
